# Changelog

## v1.1.0

当前版本聚焦于界面统一、审稿流程增强、投稿卡片修复和部署稳定性修复。

### Added

- 新增官方 Logo 与 favicon：`public/logo.svg`、`public/favicon.svg`。
- 新增审稿时间线“距今”虚拟行，显示最后一次状态更新到今天的间隔和首投至今累计天数。
- 新增成果归档字段：DOI、见刊信息、引用格式、期刊官网、APC/期刊备注。
- 新增个人统计页模块显示/隐藏，并持久化到浏览器本地。
- 新增统一设计系统层，规范卡片、按钮、输入框、胶囊和弹窗视觉语言。

### Changed

- 重构投稿卡片布局，修复长期刊名、长标题、作者列表、版本链和日期被裁剪的问题。
- 调整卡片信息层级：标题优先，作者完整显示，次级信息使用更清晰的省略和换行策略。
- 重构个人统计页，只保留投稿相关统计，移除附件完整率等管理类指标。
- 优化编辑投稿记录弹窗的字号、输入框高度、时间线显示和整体可读性。
- 更新 README、package metadata、页面 title、description、favicon 和 Open Graph 信息。

### Fixed

- 修复 GitHub Pages 部署 workflow 在缺少 `package-lock.json` 时使用 `npm ci` 导致失败的问题，改为 `npm install`。
- 修复 Header Logo 仍显示旧 `SH` 方块、`Hub` 未使用渐变的问题。
- 修复灰色文字对比度过低、状态胶囊不明显的问题。
- 修复审稿时间线仍在审状态下无法直观看到距今天数的问题。

## v1.0.0

初始版本。包含投稿记录管理、基础卡片展示、状态统计、个人统计、JSON 导入导出、Supabase 数据存储和 GitHub Pages 部署。
